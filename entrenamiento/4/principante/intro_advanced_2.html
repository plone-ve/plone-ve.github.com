Parte 1 en pocas palabras¶

El caso de estudio¶

Para este entrenamiento construiremos un Sitio Web para una conferencia ficticia de Plone.

Las principales características serán:

  • Como un visitante, yo quiero encontrar información actualizada en la conferencia.

  • Como visitante, yo quiero registrarme para la conferencia.

  • Como visitante, yo quiero ver las charlas y ordenarlas por mis preferencias.

  • Como un ponente, yo quiero ser capaz de enviar charlas.

  • Como un ponente, yo quiero ver y editar mis charlas enviadas.

  • Como miembro del jurado, yo quiero votar en las charlas.

  • Como miembro del jurado, yo quiero decidir, cuales charlas aceptar, y cuales no.

  • Como organizador, yo quiero ver una lista de todas las charlas propuestas.

Tenga en cuenta que todos nuestros requisitos conectar roles con sus capacidades. Esto es importante porque vamos a querer limitar las capacidades de aquellos a los que asignamos roles particulares.

La Anatomía de Plone¶

Capítulo completo: La Anatomía de Plone

  • Zope es un framework de aplicación Web el cual Plone corre encima de él.

  • ZODB es la base de datos de objetos utilizada

  • CMF (Content Management Framework) es un complemento para Zope para construir sistemas de gestión de contenido (como Plone).

  • Zope Toolkit / Zope 3 fue originalmente creada como una profunda re-escritura de Zope.

  • Plone usa partes provistas por el Zope Toolkit (ZTK).

  • La Zope Component Architecture (ZCA) es un sistema el cual permite por la versatibilidad y despacho de aplicaciones. Es basada en objetos que implementan una interfaz (una descripción de una funcionalidad). Pyramid, y el servidor de aplicaciones Web Python independiente, utiliza la ZCA “bajo la capucha” para ejecutar despacho de vistas y otras tareas de configuración de aplicación.

Iniciando y Deteniendo Plone¶

Tenemos el control de Plone con un pequeño script llamado “instance”:

$ ./bin/instance fg
$ ./bin/instance start
$ ./bin/instance stop
$ ./bin/instance -O Plone debug
$ ./bin/instance -O Plone run myscript.py
$ ./bin/instance adduser

Valla a la dirección URL http://localhost:8080/Plone para ver el sitio que viene con la base de datos proporcionada.

Usuarios¶

Vamos a crear un usuario de Plone que usamos para la mayor parte del entrenamiento:

  1. admin > Configuración de sitio > Usuarios y Grupos

  2. Add a user training with password: training. Add him to the group Administrators Añada un usuario llamado training con la contraseña: training. Agregarlo al grupo Administradores

Ya tenemos los siguientes usuarios en la Base de datos proporcionada:

  1. “editor:editor” (grupos: Ninguno)

  2. “jurymember:jurymember” (grupos: Ninguno)

  3. “testuser:testuser” (grupos: Ninguno)

  • El servidor de correo ya está configurado. Por favor, no abuses de esto. Lo desactivaremos después del entrenamiento.

La Interfaz de Usuario Plone¶

Tipos de Contenido¶

Estos son los tipos de contenidos por defecto:

  • Documento

  • Noticia

  • Evento

  • Archivo

  • Imagen

  • Enlace

  • Carpeta

  • Colección

Nota

Por favor, tenga en cuenta que usamos plone.app.contenttypes para la formación. Por lo tanto, los tipos se basan en Dexterity y un poco diferente de los tipos que se encuentran en un sitio Plone 4.3.x por defecto.

Nosotros creamos una estructura del sitio:

  • Agregar carpeta “El Evento” y que contenga...

    • Carpeta “Charlas”

    • Carpeta “Entrenamientos”

    • Carpeta “Sprint”

  • En la ruta /news: Agregar una Noticia “¡Sitio Web de la Conferencia esta en linea!” con alguna imagen

  • En la ruta /news: Agregar una Noticia “¡Enviar sus charlas!”

  • En la ruta /events: Agregar un Evento “Fecha limite para enviar charlas” Fecha 10.10.2014

  • Agregada una carpeta llamada “Registro”

  • Eliminado la carpeta llamada “Members” (Usuarios)

  • Agregada carpeta llamada “Intranet”
